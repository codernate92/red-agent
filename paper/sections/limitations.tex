\section{Limitations and Threats to Validity}
\label{sec:limitations}

The strongest limitation is coverage scope. The curated registry improves
reliability, but it narrows the evaluated model universe to 11 aliases.
Provider families not included in the curated registry are outside this paper's
empirical claims.

Second, the current run emphasizes campaign-native metrics and aggregate risk
rather than a dedicated evaluator-judge analysis. That choice reduces run
complexity and API spend but limits claims tied to external judge rubrics.

Third, the probe set is framework-native. The 20 ATT\&CK-inspired probes are
operationally useful, but they are not identical to HarmBench or public
\textsc{StrongREJECT} benchmark prompts.

Fourth, the engine package is a systems contribution, but the empirical results
reported here come from the stable comparison-runner path.

Finally, the size analysis remains exploratory because family-level
size-annotated coverage is still uneven.
