\section{Introduction}
\label{sec:introduction}

Large language model deployment now depends on adversarial evaluation. Early
red-teaming pipelines relied heavily on hand-authored probes, but recent work
shows that language models can generate or scale attack sets effectively
\citep{perez2022redteaming,radharapu2023aart}. In parallel, jailbreak and prompt
injection research demonstrates that alignment failures persist across diverse
attack styles \citep{wei2023jailbroken,zou2023universal,yu2024dontlisten}.

That shift creates a practical systems problem. In live evaluations, the hard
part is often not writing one more jailbreak prompt; it is running a reliable,
traceable, cross-provider pipeline that distinguishes refusals from execution
failures and emits outputs suitable for engineering decisions.

As agent capabilities improve and diffuse rapidly, robust red teaming becomes
more urgent, not less. Frameworks for cross-model jailbreak and risk evaluation
therefore need continuous updates to reflect current attack techniques and
deployment contexts. Scope discipline should not exclude domains sometimes
treated as ``lower risk'' (for example misinformation/disinformation workflows,
mental-health-adjacent interactions, or sycophancy-like behavioral failures),
because these areas can still expose meaningful weaknesses that affect future
testing design, evaluation priorities, and broader risk-assessment frameworks
\citep{perez2022redteaming,radharapu2023aart,mazeika2024harmbench}.

This paper studies \system, a multi-provider framework organized around four
ideas:
\begin{itemize}
\item provider-agnostic target adapters with JSONL trajectory logging,
\item an ATT\&CK-inspired LLM probe taxonomy and campaign runner,
\item analysis/reporting pipelines for CSV, plots, and LaTeX artifacts, and
\item a vulnerability engine for recon, focused attack, and confirmation.
\end{itemize}

This paper evaluates the framework's stable campaign-execution path, not the
newer recon/confirm engine loop. Accordingly, claims are limited to results
produced by the archived comparison run and post-hoc evaluator overlays on
those same traces. The engine is included as a systems extension and roadmap
for subsequent empirical studies.

The empirical basis is the current archived artifact set under
\path{reports/publication_run/}. The reported run executes the full 20-probe
campaign over an 11-model curated registry and includes preflight status for
all configured aliases. This provides a narrower but operationally consistent
comparison set than a broader registry with persistent credential or access
blockers.

The paper makes four contributions. First, it documents a practical
multi-provider red-teaming architecture rather than a single benchmark script.
Second, it provides an artifact-backed model comparison with unified reporting
outputs. Third, it demonstrates transparent operational accounting by pairing
campaign outcomes with provider-status artifacts. Fourth, it packages the full
pipeline for reproducibility in manuscript-ready form.
